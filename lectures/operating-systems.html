Operating Systems

Accounting Architecture

Software is the most important block in the Technology section. Software performs all of the automated tasks and facilitates all of the manual tasks in the information system. Software stores and retrieves data; software analyses and generates reports. Without software, none of the other technology components could serve their intended purpose.

Operating System

The most fundamental piece of software is the operating system (OS), which provides the user and user applications access to the computer hardware. Usually, operating systems comprise a kernel and userland. Additionally, operating systems generally require two supporting tools: a firmware interface and a bootloader.

At the Heart of the Matter

The kernel is the part of the operating system that interacts with the computer hardware. This is the lowest level of the operating system. The kernel receives requests from user applications to access computer hardware, such as the CPU, RAM, storage, audio, or video, and forwards these requests to the hardware components. Once the request is fulfilled, the kernel returns the results of the request to the application. For example, if the user wishes to retrieve a spreadsheet from storage, the spreadsheet application requests the file from the kernel. The kernel retrieves the file from storage and returns the file to the application. (This process has additional steps, such as transferring the file to main memory (i.e., RAM), but this simplified description is sufficient to explain the role of a kernel.)

Types of Kernels

Depending on the type of kernel, the kernel itself may communicate directly with user applications (monolithic kernel) or the kernel may communicate with a series of servers that serve as intermediaries between the kernel and user applications (microkernel). Either way, the kernel interacts with the hardware. Some praise microkernels for their simplicity, and others prefer monolithic kernels because they handle requests more quickly.

Daemons

Frequently, the word “server” is used to identify a specific type of computer hardware, but a more appropriate definition would be software that provides a service, also known as a daemon. Unlike more visible interactive applications, daemons run in the background without user input, and many start when the operating system boots. Micro kernels use daemons to communicate with user applications, but operating systems with monolithic kernels also use daemons to handle several functions, such as network communication, user input, and output to computer peripherals, such as monitors and speakers. Some of these daemons are labeled “middleware” because they sit between the kernel and user applications.

Userland

Userland is the part of the operating system that interacts with the user. The primary layer in userland is the shell, which provide the user either a text-based user interface, commonly referred to as a terminal or a console, or a graphical user interface (GUI). Almost all personal computing devices have graphical shells, but the vast majority of server computers only have terminals. These shells receive commands from the user, and they provide user applications when requested.

Putting It All Together

Firmware Interface

The kernel and userland make an operating system, but in order for the operating system to be available to the user, a firmware interface is necessary. Two common forms are Basic Input/Output System (BIOS) or, more recently Unified Extensible Firmware Interface (UEFI). This firmware resides on the motherboard's own flash storage, and it performs two important functions. The first is to initialize and test the computer hardware, and the second is to load the bootloader.

BIOS and UEFI are only two examples of firmware. Firmware comprises a large variety of embedded software that controls hardware. For example, jets and space shuttles use firmware to control their systems, and the same applies to thermostats, video game systems, and automobiles.

Bootloader

The firmware interface initializes the storage drive and looks for a bootloader on the drive. The bootloader contains the instructions necessary to locate and boot the operating system kernel. Although this process is frequently transparent to Windows and OS X users, some operating systems expose the bootloader to the user so that the user can customize how the operating system boots.

Which Operating Systems?

Microsoft Windows and Apple OS X are the two dominant desktop and laptop operating systems, but these are effectively irrelevant in other components of the information system (e.g., servers, mobile devices, firmware). Although less well known, Unix, BSD, and Linux run the overwhelming majority of information system hardware. As a result, the ability to participate in systems design and maintenance relies in large part on familiarity with these operating systems. Familiarity is not equivalent to knowledge of the historical background, but because these three operating systems, their developers, and their philosophies have had, and continue to have, such a profound impact on the software community, especially open source software, a brief history is warranted.

UNIX

1969: Researchers at AT&T Bell Laboratories became dissatisfied with Multics, the operating system that they had begun to develop with MIT and General Electric, and they decided to develop an alternative operating system, which they named UNIX.

Unix engineers operated under an implicit philosophy, which continues to govern software development practices: Write programs that do one thing and do it well. One popular representation of this philosophy is KISS (Keep It Simple, Stupid!) which serves as a reminder that simplicity is more virtuous than complexity.

Licensing

1973: AT&T began selling UNIX licenses—AT&T was a regulated company and could only sell licenses and not software—to universities. Unlike common current software licenses, UNIX licenses included source code and no other support. The availability of source code, the lack of commercial support, and the distribution of licenses to universities motivated the development of communities that supported, modified, and improved on the operating system.

Microsoft Who?

1978: Microsoft purchased a UNIX license and began creating Xenix for what Bill Gates believed was the impending proliferation of personal computers. Microsoft eventually abandoned Xenix in favor of MS-DOS, but the existence of Xenix is evidence of a very important point: Unix, and other subsequent Unix-like operating systems, are not merely alternatives to Microsoft Windows. Unix was the operating system of choice before anyone had heard of Microsoft.

System V

1983: Bell Systems broke up after an antitrust case, and AT&T became able to sell Unix. AT&T released a commercialized Unix System V and forbade access to Unix source code, which essentially ended community Unix development.

Knowing no other model than secrecy for collecting profits from software and no other model than centralized control for developing a commercial product, AT&T clamped down hard on source-code distribution.
– Eric Raymond

BSD

1974-1980: Before AT&T stopped distributing Unix source code, the University of California, Berkeley purchased a license and began modifying the operating system. Berkeley subsequently released their modifications as a separate operating system, which they called BSD (Berkeley Software Distribution). Through the fourth release of BSD (4BSD), users needed to have purchased a UNIX license from AT&T. Interestingly, many licensees, including employees of Bell Labs, preferred 4BSD to System V, and the Defense Advanced Research Project (DARPA) funded the creation of the Computer Systems Research Group (CSRG) at Berkeley to develop a Unix for ARPAnet, the precursor to the Internet.

An Open Standard

Despite the appeal of both System V Unix and BSD Unix, disputes arose between the programming communities. In an attempt to reconcile differences, the Institute of Electrical and Electronics Engineers (IEEE) developed a set of Portable Operating System Interface (POSIX) standards based on System V Unix which continue to govern Unix and Unix-like operating systems today. The list of POSIX-certified operating systems currently contains the three most prominent Unix offerings: IBM AIX, HP-UX and Oracle Solaris. Although BSD continues not to be POSIX-certified, it is mostly POSIX compliant.

BSD Becomes Free

1992: CSRG began replacing the remaining AT&T code in 4BSD with code that they had written. Once CSRG had removed all AT&T code, it released 386BSD as a free Unix-compatible operating systems for Intel's 80386 processor, an early iteration of the x86 line of CPUs. Communities began to develop two different versions of 386BSD: NetBSD for technical users and FreeBSD for less technical PC users. As a result, of releasing 386BSD for free, AT&T sued Berkeley claiming that the operating system contained proprietary code.

After the Suit

Although Berkeley was exonerated, the legal uncertainty during the two-year suit encouraged many developers to transition to Linux. The BSD community has been much smaller ever since, but BSD is still widely used and supported, and many flavors exist. The most prominent are FreeBSD, NetBSD, OpenBSD (focused on security) and Mac OS X. Furthermore, most operating systems, including Microsoft Windows, use network tools from BSD. In 2006, InformationWeek rated 4BSD as the “Greatest Software Ever Written” and the “Theoretical undergirder of the Internet.”

Linux

Linux is currently the most widely adopted operating system. It began in 1991 when Linus Torvalds, a Finnish university student, decided to develop a free Unix-compatible kernel from scratch as a hobby. Today, 70% of internet servers, 55% of mobile devices and 97% of supercomputers run Linux. It is also the operating system of choice for many single-board computers, such as Raspberry Pi. Although the Linux kernel is one project, the resulting operating system has over 600 different versions (called distributions or distros, for short). Some of the more prominent distributions are Debian GNU/Linux, Ubuntu Linux, Red Hat Enterprise Linux, and Android. Each of these uses a different combination of kernel and userland tools to provide a specialized user experience.

GNU/Linux

In actuality, Linux is not an operating system. It is merely the operating system kernel. Many of the userland tools come from a similar, but essentially failed, attempt by none other than Richard Stallman to develop a free Unix-compatible operating system called GNU. Because the userland tools come from the GNU project, Richard Stallman insists that the Linux operating system should actually be called GNU/Linux to highlight the contributions of both projects. The Debian GNU/Linux distribution has adopted this name, but most others prefer the shorter label.

Accounting Tie-in

The operating system is the most foundational piece of software. As a result, it is important for accounting architects to understand which operating system options are available for designing information systems, and traditional auditors and IT auditors must understand operating systems in order to audit existing systems. Enterprise-grade information systems run on Unix, BSD, and Linux, and audits face significant roadblocks when no auditors on the team have ever even heard those names before.
