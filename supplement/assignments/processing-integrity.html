Interactive Assignment

Database normalization is necessary to ensure data processing integrity. This assignment will give you practice normalizing a non-normalized database. Imagine that your client is a radio station. The client has given you a few records from their non-normalized database, and you need to convert the single non-normalized table into four tables that satisfy third normal form (3NF). The next screen displays the snippet of the client's database. Follow these steps to prepare a correct solution:

  1. Divide the attributes into four separate tables to resolve all dependencies and satisfy 3NF.
  2. Identify and label the primary key for each table. Do not pick an attribute as primary key that may be duplicated (e.g., more than one song may have the same name). If necessary, create a new attribute with an ID for each record in the table.
  3. Include the primary key from one table as a foreign key in another table to link the data in each table together.

Create your normalized tables in a spreadsheet program with primary and foreign keys correctly labeled. Export your spreadsheet to PDF, and upload it to Dropbox on eCourseware.
