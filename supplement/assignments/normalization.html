Normalization

Database normalization is necessary to ensure data processing integrity. This assignment will give you practice normalizing multiple non-normalized databases. Each of the next screens has either a single non-normalized database or an E-R diagram of a non-normalized database. Some of the databases on the following screens are in the lecture, and you view the set of normalized tables in the lecture after you have created your own in order to verify its accuracy and completeness. However, this assignment will not be useful to you in learning to normalize databases unless you draft your solution first and only look at the solution in the lecture afterwards.

Steps

Follow these steps for each screen in order to identify a set of tables that satisfy third normal form (3NF) for each database:

  1. Divide the database attributes into separate tables to resolve all dependencies and satisfy 3NF. No need to draw anything fancy, you can simply create lists of attributes (without any cell values), one list for each normalized table with a table name at the top of the list.
  2. Identify and label the primary key with a PK for each table. Do not pick an attribute as primary key that may be duplicated. If necessary, create a new attribute with an ID for each record in the table.
  3. Link the normalized tables together by including the primary key from one table as a foreign key in another table and label it with an FK.

Create your normalized tables in a word processor document. Export your document to PDF, and upload it to Dropbox on eCourseware.
