Version Control

GitHub is a popular cloud-based store of git repositories. In fact, the materials for this course are stored on GitHub. This assignment will teach you how to use both git and GitHub. Git is a VCS, and GitHub is a Web service that hosts git repositories. I have created a repository on GitHub, and you will use git to make contributions to this repository following the steps on the next screens. The Availability lecture appendix and this article explain how to use GitHub and git on Linux.

  1. Create a free GitHub account. (link)
  2. Log in to GitHub, navigate to the assignment repository page (link), and click the ‘Fork’ button on the right-hand side of the screen. Clicking the fork button will create a fork (i.e., copy) of the assignment repository in your GitHub account.
  3. Clicking the fork button should also navigate you to the page of your fork of the assignment repository. Ensure that you are on the page of your fork of the assignment repository. If you are not on the page of your fork, navigate to your fork, which will be at the URL https://github.com/USERNAME/github-practice (replace USERNAME in the URL with your GitHub user name).
  4. Click the ‘Code’ button and copy the URL under the label ‘Clone with HTTPS.’

Instructions - Page 1 of 3

  1. Use git in an SSH Terminal of your Linux container to clone your fork using the URL from the previous step.
  2. List the contents of your current folder to confirm that you have cloned the repository and created a folder named github-practice.
  3. Navigate to the Quotes folder within the github-practice folder.
  4. Using the UNIX philosophy, redirect your favorite quote to a file with your name. For example, my quote would be “When we speak of free software, we are referring to freedom, not price. --Richard Stallman” and my filename would be ProfessorCoyne. (NB: Do not use any spaces in your filename.)
  5. List local repository changes to confirm that you have an untracked file with your name.

Instructions - Page 2 of 3

  1. Stage your changes.
  2. Assign yourself a username in git: git config user.name. (For example, I would type: git config user.name "Professor Coyne").
  3. Assign yourself an email address in git: git config user.email. (For example, I would type: git config user.email "jgcoyne@memphis.edu").
  4. Commit your changes to your local git repository with the commit message "Add [Your Name] quote" (without brackets). For example, my commit message would be "Add Professor Coyne quote".
  5. Print the name of the current branch. (You will use this branch name to push your changes.)
  6. Print the name of the cloud repository. (You will use this repository name to push your changes.)

Instructions - Page 3 of 3

  1. Push the changes in your current branch to the GitHub repository using the branch name and cloud repository named printed previously. Git will ask for your GitHub username and password. (When you type your password, your container will not provide any visual feedback of your typing, but it will record your key presses, so simply type your password and press ENTER.)
  2. After pushing your changes, return to your forked repository on GitHub.
  3. Click the gray ‘New pull request’ button.
  4. On the next two screens, click the green ‘Create pull request’ button. After clicking this button on the second screen, you may close the browser window.

Once you have completed those steps, I will receive a notification from GitHub. This will serve as evidence that you have completed the assignment.
