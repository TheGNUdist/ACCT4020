Frequently Asked Questions - Page 1

  1. Q: I never received a prompt to set the root password when I installed MySQL. What is the password?
    A: You never received a prompt because the terminal window was too small during the installation process. As a result, the root user has no password, and you can access your database with the command: “mysql -uroot” (without quotes).
  2. Q: When I try to log in to the database, I receive the error: “Access denied for user ‘root’@‘localhost’.” What should I do?
    A: You must prepend sudo to your mysql command: sudo mysql -uroot (if the root user has no password) or sudo mysql -uroot -p (if the root user has a password). If you continue to have this error after adding sudo, then you have entered the wrong password.

Frequently Asked Questions - Page 2

  1. Q: When I try to log in to the database, I receive the error: “Can't connect to local MySQL server through socket ‘/var/mysql/mysql.sock’” What should I do?
    A: This error means that the database server has not started. First, try restarting the virtual machine. If that fails to resolve this issue, then you must uninstall and reinstall MySQL.
  2. Q: What is my password?
    A: Contrary to popular opinion, I cannot answer all technical questions, and I can definitely not tell you what password you set while installing MySQL. If you believe that you have not set a password during install, then please consult question (2).
