Project - Intro

This assignment tests your ability to query a database to analyze data. Your assignment is to identify the queries necessary to find the answers.

Begin by using apt to install mysql-server on your Linux container. (You should already have installed the dialog and man-db packages previously. These are necessary to install mysql-server. During the install, leave prompts to create a password blank by simply pressing ENTER.) Follow the steps on the next slide of these instructions to create and import your database. Use SQL to create queries to answer the questions on the final slide.

Take a screenshot of as much of the output of each query as possible, and paste each screenshot into a single document along with the query used to create the output. Export the document to PDF, and upload it to Dropbox on eCourseware. I will grade solely on the accuracy of the queries.

Linux Commands

  1. Verify that MySQL server is running: sudo service mysql start
  2. Log in to your MySQL database server: mysql -uroot
  3. Create a database named company: create database company;
  4. Exit MySQL: quit;
  5. Use the following command to download the database file from the course portal:
    wget -O company.sql https://goo.gl/aQwKm3
    (Note: That is a capital O after the hyphen, not a zero.)
  6. Import the Database file into MySQL:
    mysql -uroot company < company.sql
  7. Log in to the newly created database: mysql -uroot company. From this screen, you can enter SQL queries. Use all capital letters in SQL queries.
    (e.g., SELECT ID, USERNAME FROM EMPLOYEE;)

MySQL Commands

Use of MySQL for data retrieval and analysis requires knowledge of both SQL and some built-in MySQL commands. (NB: Although it is best to type SQL commands in all uppercase letters, these commands are best executed in lowercase letters because they are case-sensitive and MySQL database names are commonly in lowercase letters. Also, just like SQL queries, all MySQL commands should end in a semicolon.)

Command
Explanation
show databases;
Print a list of all databases
create database homework;
Create a new, empty database and name it homework
use company;
Execute all future commands in this MySQL session against the company database
show tables;
Print a list of all tables in the currently selected database
describe CUSTOMER;
Print a list of all fields in the CUSTOMER table with their data types
quit;
Exit MySQL

Tips

Begin each question by looking at the Dictionary file for the relevant table(s). The Dictionary file on the course portal provides a list of all tables and attributes with their respective definitions. Then, use MySQL to print out the table in its entirety: SELECT * FROM [TABLENAME]; (Replace [TABLENAME] with the actual table name. For example, SELECT * FROM VENDOR; to view all data in the Vendor table.) The data may be more than you can see on one screen, but you can use your mouse to scroll in Codeanywhere.

Before considering SQL syntax, think logically about what you want to know. This will help you to determine which attributes to select (e.g., Do I need ID or PO?), which summary functions may be necessary (e.g., Do I need to add values from multiple records?), and how to limit the results to records that match all question criteria (e.g., I only need orders in March.)

Questions

  1. (PAYMENT table) Which payment IDs have no purchase order number?
  2. (SALE table) What is total sales revenue?
  3. (EMPLOYEE table) How many employees work at each location? (Your query should report one number for each location.)
  4. (RECEIPT and PURCHASE tables) Which, if any, receipt has a different quantity of raw materials delivered than were ordered in the purchase order?
  5. (SALE table) Which customer(s) placed the largest single orders?
