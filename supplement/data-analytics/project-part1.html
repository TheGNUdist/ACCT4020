Project Part 1 - Intro

This analytics project will give you practical experience using a relational database management system to analyze data. For part 1 of this project, you will install a database server, import a database, run queries, and report your results. Pretend that a client has given you access to a dataset and a Dictionary file, which you can find on the course portal, that explains the dataset. The client wants you to answer a number of questions using SQL queries. The next screen explains how to install the relational database management system MySQL. The screen screen provides a number of MySQL commands that you will need for this project. The third screen explains how to create and import the client's database. The final screen lists the questions and queries and explains how to report your results.

Install MySQL

  1. Prerequisite: Your container must have the dialog and bsdmainutils packages installed. These should be installed by default in all Blank Ubuntu 16.04 containers. If you are uncertain whether these are already installed, try to install them using your package manager. If these packages are already installed, your package manager will respond that they are already the newest version.)
  2. Install mysql-server. During the install, you will see a dialog window asking for a password for the database root user. Enter a password that is easy for you to remember. You will not receive a visual feedback as you type the password (i.e., the screen will not move the cursor or show any asterisks to represent the password characters). Press ENTER when you have finished typing the password. On the next dialog screen, type your password again to confirm and press ENTER.
  3. Start the MySQL server: sudo systemctl start mysql.service

MySQL Commands

Use of MySQL for data retrieval and analysis requires knowledge of both SQL and some built-in MySQL commands. (NB: Although it is best to type SQL commands in all uppercase letters, these commands are best executed in lowercase letters because they are case-sensitive and MySQL database names are commonly in lowercase letters. Also, just like SQL queries, all MySQL commands should end in a semicolon.)

Command
Explanation
show databases;
Print a list of all databases
create database homework;
Create a new, empty database and name it “homework”
use company;
Execute all future commands in this MySQL session against the company database
show tables;
Print a list of all tables in the currently selected database
describe CUSTOMER;
Print a list of all fields in the CUSTOMER table with their data types
quit;
Exit MySQL

Create and Import Database

  1. Log in to your MySQL database server: mysql -u root -p (Enter the root user password that you created when installing mysql-server into the password prompt. For security reasons, there will be no visual feedback as you type the password. Press ENTER after typing the password.)
  2. Create a database and name it “company” (without quotes).
  3. Exit MySQL.
  4. Download the database file from the following URL to your container and name it “company.sql” (without quotes): https://goo.gl/Mc5Czc
  5. Import the Database file into MySQL:
    mysql -u root -p company < company.sql
  6. Log in to the newly created database: mysql -u root -p company. From this screen, you can enter SQL queries. Use all capital letters in SQL queries.
    (e.g., SELECT ID, USERNAME FROM EMPLOYEE;)

  1. Which sale approvals have no approval date?

    2. SELECT ID FROM SALE WHERE APPROVALDATE = 0000-00-00;

  2. What is the total we have paid to vendors for the current period?

    3. SELECT SUM(AMOUNT) FROM PAYMENT;

  3. How many customers does each sales rep have?

    4. SELECT SALESREP, COUNT(ID) FROM CUSTOMER GROUP BY SALESREP;

  4. Did I underpay any invoices?

    5. SELECT INVOICE.ID FROM PAYMENT JOIN INVOICE
    ON PAYMENT.INVOICE = INVOICE.ID WHERE PAYMENT.AMOUNT <
    INVOICE.QUANTITY * INVOICE.UNITCOST + INVOICE.FREIGHTCOST;
