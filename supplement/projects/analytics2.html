Install MySQL

  1. Verify that your container has the dialog and logger applications installed. Install them, if necessary. (NB: Logger is part of the bsdutils package.)
  2. Search for the MySQL database server metapackage.
  3. Install the MySQL database server metapackage. During the install, you will see a dialog window asking for a password for the database root user. Enter a password that is easy for you to remember. You will not receive a visual feedback as you type the password (i.e., the screen will not move the cursor or show any asterisks to represent the password characters). Press ENTER when you have finished typing the password. On the next dialog screen, type your password again to confirm and press ENTER.
  4. Use systemd to check the status of the MySQL server.
  5. Start the MySQL server.
  6. Enable the MySQL server to launch when you boot the container.

Create and Import Database

  1. Log in to your MySQL database server as the root user. When prompted, enter the root user password that you created when installing mysql-server into the password prompt. For security reasons, there will be no visual feedback as you type the password. Press ENTER after typing the password.
  2. Create a database and name it “company” (without quotes).
  3. Exit MySQL.
  4. Download the database file from https://bit.ly/2Zq9qsq to your container and name it “company.sql” (without quotes).
  5. Import the database file into MySQL by using the UNIX philosophy to redirect the company.sql file to the MySQL login command. Use the root user and the name of the database that you created in the login command. (NB: Mysql is the application and company.sql is the file.)
