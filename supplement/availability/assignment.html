Git and GitHub Assignment

Version control systems are a useful backup and collaboration tool. Git is currently perhaps the most widely used VCS, used for, among other things, the source code for the Linux kernel. GitHub is a online tool that stores Git repositories to promote broader code sharing. In fact, the materials for this course, except for the quizzes, are on GitHub. This assignment will teach you how to use both. I have created a repository on GitHub, and you will use Git to make contributions to this repository using these steps:

Steps

  1. Create a free GitHub account. (link)
  2. Fork the GitHub repository. (link)
  3. Install Git in your container. (sudo apt-get install git)
  4. Clone your fork of the GitHub repository.
  5. Add a text file with a quote of your choosing and your name as the title to the AddYourNameFileHere folder.
  6. Add your name and the date you completed this assignment to the list.txt file.
  7. Stage, commit, and push your changes to GitHub.
  8. Initiate a pull request to have your changes merged into the original GitHub repository.
