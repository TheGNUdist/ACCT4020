Practice - Intro

GitHub is a popular cloud-based store of git repositories. In fact, the materials for this course are stored on GitHub. This assignment will teach you how to use both git and GitHub. Before you begin, your container must have two necessary applications: git and nano. Git is the VCS that you will use for this assignment, and nano is a text editor. Use these steps to confirm that git and nano are installed in your container:

  1. Check that git is installed: which git. (If this command outputs the path to git, e.g., /usr/bin/git, then git is installed, if the command outputs nothing, then git is not installed.)
  2. Check that nano is installed: which nano.
  3. Use apt to install whichever package is missing.

I have created a repository on GitHub, and you will use git to make contributions to this repository following the steps on the next screens:

Instructions - Page 1 of 3

  1. Create a free GitHub account. (link)
  2. Log in to GitHub, navigate to the assignment repository (link), and click the ‘Fork’ button in the top right corner of the screen.
  3. Click the ‘Clone or download’ button and copy the URL under the label ‘Clone with HTTPS Use Git or checkout with SVN using the web URL.’
  4. Use git in your container to clone your fork using the URL from the previous step. (Review the Availability lecture for help with git commands.)
  5. Navigate to the Quotes folder within the acct4020-github-assignment folder. (Review the Storage lecture for help with this command.)
  6. Use nano to edit a new text file with the filename FirstnameLastname (e.g., nano ProfessorCoyne).
  7. Type your favorite quote (e.g., mine is “When we speak of free software, we are referring to freedom, not price.” --Richard Stallman).

Instructions - Page 2 of 3

  1. Once you have finished typing your quote, save your changes with CTRL-X, then press ‘y’ and ENTER.
  2. List local repository changes to confirm that you have an untracked file with your name.
  3. Stage your changes.
  4. Assign yourself a username in git: git config user.name. (For example, I would type: git config user.name "Professor Coyne").
  5. Assign yourself an email address in git: git config user.email. (For example, I would type: git config user.name "jgcoyne@memphis.edu").
  6. Commit your changes to your local git repository. This will open a new nano window. Enter a commit message. (For example, I would type Added ProfessorCoyne quote.). Save your changes with CTRL-X, y, and ENTER, just as in the earlier step.

Instructions - Page 3 of 3

  1. Push your changes to your GitHub repository. Git will ask for your GitHub username and password. When you type your password, your container will not provide any visual feedback of your typing, but it will record your key presses, so you can simply type your password and press ENTER.)
  2. After pushing your changes, return to your forked repository on GitHub.
  3. Click the gray ‘New pull request’ button.
  4. On the next two screens, click the green ‘Create pull request’ button. After clicking this button on the second screen, you may close the browser window.

Once you have completed those steps, I will receive a notification from GitHub. This will serve as evidence that you have completed the assignment.
