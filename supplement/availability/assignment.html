Interactive Assignment - Intro

GitHub is a popular cloud-based store of git repositories. In fact, the materials for this course are stored on GitHub. This assignment will teach you how to use both git and GitHub. Git is a VCS, and GitHub is an Web service that hosts git repositories. I have created a repository on GitHub, and you will use git to make contributions to this repository following the steps on the next screens.

  1. Create a free GitHub account. (link)
  2. Log in to GitHub, navigate to the assignment repository (link), and click the ‘Fork’ button in the top right corner of the screen.
  3. Click the ‘Clone or download’ button and copy the URL under the label ‘Clone with HTTPS Use Git or checkout with SVN using the web URL.’

Instructions - Page 1 of 3

  1. Use git in an SSH Terminal of your Linux container to clone your fork using the URL from the previous step.
  2. List the contents of your current folder to confirm that you have cloned the repository and created a folder named acct4020-github-assignment.
  3. Navigate to the Quotes folder within the acct4020-github-assignment folder.
  4. Using the UNIX philosophy, redirect your favorite quote to a file with your name. For example, my quote would be “When we speak of free software, we are referring to freedom, not price. --Richard Stallman” and my filename would be ProfessorCoyne. (NB: Do not use any spaces in your filename.)
  5. List local repository changes to confirm that you have an untracked file with your name.

Instructions - Page 2 of 3

  1. Stage your changes.
  2. Assign yourself a username in git: git config user.name. (For example, I would type: git config user.name "Professor Coyne").
  3. Assign yourself an email address in git: git config user.email. (For example, I would type: git config user.email "jgcoyne@memphis.edu").
  4. Commit your changes to your local git repository with the commit message "Add [Your Name] quote" (without brackets). For example, my commit message would be "Add Professor Coyne quote".
  5. Push the changes in your local master branch to the GitHub repository named origin. Git will ask for your GitHub username and password. (When you type your password, your container will not provide any visual feedback of your typing, but it will record your key presses, so simply type your password and press ENTER.)

Instructions - Page 3 of 3

  1. After pushing your changes, return to your forked repository on GitHub.
  2. Click the gray ‘New pull request’ button.
  3. On the next two screens, click the green ‘Create pull request’ button. After clicking this button on the second screen, you may close the browser window.

Once you have completed those steps, I will receive a notification from GitHub. This will serve as evidence that you have completed the assignment.
