Proprietary is Closed

  • Copyright prohibits making, transmitting, displaying or selling copies or derivatives of a work.
  • Patent prohibits making, using or selling an invention.
  • Trademark prohibits use of a name, symbol, figure, letter word or mark adopted and used by a manufacturer or merchant.
  • Non-disclosure agreement restricts disclosure of information to external parties.
  • Non-compete clause prevents use of trade secrets in a competing enterprise.

These all restrict use of intellectual poperty.

Open Standard

A standard is a set of specifications. Although no common definition of open standard exists, the Modern Paradigm for Standards lists a number of characteristics of open standards:

  1. Cooperation between standards organizations
  2. Adherence to fundamental principles of standards development: due process, broad consensus, transparency, balance, openness
  3. Collective empowerment that promotes interoperability, scalability, stability, competition and innovation.
  4. Availability
  5. Voluntary Adoption

Standards Bodies

  • World Standards Cooperation (WSC) comprising International Organization for Standardization (ISO), International Electrotechnical Commission (IEC) and International Telecommunication Union (ITU)
  • World Wide Web Consortium (W3C)
  • Institute of Electrical and Electronics Engineers (IEEE)
  • Internet Engineering Task Force (IETF)

Prominent Open Standards

  • World Wide Web (W3C)
  • Internet protocol suite (IETF)
  • Universal Serial Bus (USB-IF)
  • Wireless Fidelity (IEEE)
  • Synchronous dynamic random-access memory (JEDEC)
  • Dublin core (DCMI)

Remember, not all widely adopted standards are open.

Open Formats

File formats are standards for the storage of digital media. Open formats rely on open standards. Popular open file formats include:

  • Images: JPEG 2000
  • Audio: Ogg, FLAC
  • Video: Matroska
  • Text: Plain text, HTML, ePub, Office Open XML, OpenDocument, CSV, XML
  • Other: iCalendar, WebDAV

Here again, not all popular formats are open.

In the Beginning

(1950s to be exact...)

Developers shared software code for free without copyright. This may have been in part the result of software being developed for a single hardware architecture, so no application could achieve wide adoption. One popular software development focus was time-sharing applications, which allowed multiple users to use the same computer hardware at the same time.

Then Came the Software License

(Turn the clock forward to 1970...)

Developers transitioned from assembly code (low-level code that was closely tied to machine architecture) to higher-level, architecture-independent code that resulted in more complicated programs. As a result, developers wanted to receive compensation for their effort. Unix, a new, portable operating system, was an early example of software that required the purchase of a license.

What Is a License?

A license allows the owner of intellectual property to grant use to others without giving up ownership.

What are some common examples of licensing that we encounter?

All Fun and Games Until Someone Gets Hurt

In 1979, Richard Stallman worked for MIT, and his department had a Xerox laser printer that would jam. The programmers in the department could have created a program that would have the printer report a jam, but they had no access to the printer's source code, and anyone who had received a copy was under a non-disclosure agreement not to share it.

And Man Created the Free Software Movement

In order to combat the power of software vendors and recreate the earlier environment of shared software, Richard Stallman set out to create a free Unix-compatible operating system called GNU. In 1985, he established the Free Software Foundation (FSF) to support the development of free software, and in 1989, he released the GNU General Public License (GPL) for releasing free software.

Four Freedoms

A program is free software if the program's users have four essential freedoms:

  • The freedom to run the program as you wish, for any purpose (freedom 0).
  • The freedom to study how the program works, and change it so it does our computing as you wish (freedom 1).
  • The freedom to redistribute copies so you can help your neighbor (freedom 2).
  • The freedom to distribute copies of your modified versions to others (freedom 3).

Still a License

It may seem odd that an opponent of proprietary software licenses should choose to introduce a software license of his own. Could these two attitudes be reconcilable?

FSF is not Alone

Since the inception of the the Free Software Foundation, other organizations have joined to promote free software. One prominent example is the Open Source Initiative (OSI). Not all groups agree on the correct approach to free software. For example, OSI believes that Richard Stallman is too extreme. He responded, “Open source is a development methodology; free software is a social movement. For the Open Source movement, non-free software is a suboptimal solution. For the Free Software movement, non-free software is a social problem and free software is the solution.”

I remain agnostic because I cannot perceive a difference in free software outcomes, but feel free to form your own opinion.

“Free” or “Open Source”?

Although free software is the focus of the FSF and open source software is the focus of the OSI, these terms are frequently used interchangeably or combined into FOSS (free and open source software).

Gratis versus Libre

Free software is free as in freedom not as in beer
(or lunch, for economists). Free software is frequently also provided at no cost, but this is a by-product, not the goal. In fact, when Richard Stallman completed Emacs, a text editor for his GNU operating system, he sold copies of the software. The copies he distributed satisfied the four freedoms, but the copy was not gratis.

Open-source Hardware Licenses

Open source hardware projects frequently co-opt open-source software licenses for their output. Unfortunately, open-source software licenses are not a great fit for hardware projects because these licenses address copyright and not patents. As a result, a few groups have developed hardware-specific open licenses:

  • TAPR Open Hardware License
  • CERN Open Hardware License

Open System

  • Interoperability allows third parties to develop compatible software or peripherals.
  • Portability allows the system to run on multiple hardware architectures.

Because of the interchangeability of parts in a traditional desktop computer, it may be tempting to conclude that towers constitute open hardware. However, open systems can comprise closed components, and it would be more appropriate to refer to these desktops as open systems. Microsoft Windows is another example. Open Access Licenses

Open access can apply to any knowledge base or creative work, not simply academic research (generally not recommended for software).

  • Creative Commons License (CC) with a combination of the following conditions:
    • Attribution (BY) requires giving the creator credit.
    • Share-alike (SA) restricts the license of derivative works to be the same as the original.
    • Non-commercial (NC) prevents the use of the work for commercial purposes.
    • No Derivative Works (ND) allows only verbatim copies.

Gratis versus Libre, Again

With open access the goal is to provide content at no cost. In this case, gratis and libre both mean free as in beer.
